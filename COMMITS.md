# Commits Guidelines

The commit messages in this project should follow the [Conventional Commits](https://www.conventionalcommits.org/es/v1.0.0/) specification. This helps maintain a clear and consistent commit history, making it easier to understand the changes made to the codebase and to automate release processes.

The types of commits are based on the [Angular convention](https://github.com/angular/angular/blob/22b96b9/CONTRIBUTING.md#-commit-message-guidelines).
